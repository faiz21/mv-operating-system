# Workflow Blueprint

## Position in the operating model
This skill sits above ordinary task skills. It is a meta-skill used to create new standardized skills.

## Upstream inputs
- user request to create a new skill
- workflow summary or task domain
- constraints, approvals, and required outputs

## Downstream outputs
- completed skill package
- reusable authoring pattern for future skills

## Role interaction map
- Main Company drafts the initial package
- Auditor checks structural compliance and anti-bloat rules
- Consultant improves maintainability, routing clarity, and reference design
- Human owner approves final standards or publication decisions

## Human vs AI split
- AI may scaffold, draft, route, and structure the package
- Humans approve standards-level decisions, publication, and governance changes

## Handoff map
1. intake the skill request
2. draft package structure
3. route by role for review if needed
4. revise
5. hand off for approval or use

## Escalation path
Escalate to the user when workflow semantics, role boundaries, or approval requirements are unclear.
