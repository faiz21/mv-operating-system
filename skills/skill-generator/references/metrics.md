# Metrics

## KPI definitions
- Entry point size: `SKILL.md` should remain compact relative to total package detail
- Routing clarity: each relevant role can find its next file in one pass
- Reference precision: each reference file has a distinct concern
- Duplication rate: repeated doctrine across files should be minimal

## Review triggers
- `SKILL.md` becomes noticeably bloated
- `agents.md` grows into a second encyclopedia
- multiple reference files overlap heavily
- users repeatedly need clarification on where logic lives

## Reporting cadence
- review on initial creation
- review on every major revision
- review whenever the skill standard itself changes
