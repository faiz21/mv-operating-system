# Redesign Guidelines

## Pain points to watch
- oversized `SKILL.md`
- weak role routing
- references split too finely or too coarsely
- specialist sections added without real need
- duplicated instructions across files

## Optimization questions
- can `SKILL.md` be shortened without losing executable clarity?
- does `agents.md` route clearly without becoming another bloated doctrine file?
- are reference files grouped by concern rather than by convenience?
- are optional files actually necessary?

## Anti-patterns
- stuffing every rule into the entrypoint
- forcing all four views even when not materially relevant
- creating empty standards theater
- using examples as the only source of required logic

## Improvement levers
- move doctrine out of `SKILL.md`
- keep routing concise and role-specific
- keep references single-purpose
- use scripts for deterministic operations instead of prose when appropriate
