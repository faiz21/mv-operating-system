# Audit Checklist

## Structural checks
- `SKILL.md` exists and is the canonical entrypoint
- `agents.md` exists and routes relevant roles correctly
- reference files exist only when needed
- optional directories are used only when justified

## Entry point checks
- `SKILL.md` is compact
- objective is clear
- inputs and outputs are explicit
- execution logic is actionable
- escalation rules are present
- reference map points to real files

## Routing checks
- each relevant role has a clear reading path
- role-specific decisions are bounded
- prohibited assumptions are explicit
- escalation routing is defined

## Failure conditions
- bloated `SKILL.md`
- duplicated heavy doctrine across files
- fake or empty reference files
- invented business rules or governance
- unclear role ownership

## Prohibited shortcuts
- using `SKILL.md` as a dumping ground
- creating references with no real purpose
- hiding critical logic only in examples
- collapsing execution, audit, and consulting responsibilities into one role
